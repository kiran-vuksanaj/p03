What is a Graph?

According to the Oxford dictionary, a graph is a diagram showing the relation between variable quantities. As we know, there are many different types of graphs, such as bar graphs, line graphs, histograms, and pie charts.

2

What is a Graph Database?

A graph database is a collection of nodes, each representing its own entity, that have defined relationships with each other. Below is an example of a very simple graph database.

3

What are graph databases used for?

Graph data bases are used for many different situations.For example, Walmart used a graph database to implement their reccomendation engine based on the relationships with past searches. Also, the US Army uses a graph database to track data about their machinery so they know when they need to replace parts. Graph databases are also very useful in fraud detection.

4

Pros and Cons

Pros
  • Flexibility in how you define your nodes and relationships
  • Fast relationship-based search and indexing
  • Graph databases have specific functionalities to store/manage graphs
Cons
  • Not always efficient based on the relationships between data pieces
  • All data stored on one server so not very scalable
  • Can't perform mass analytics
